Основы программирования

Сам по себе процесс программирования можно охарактеризовать как общение с компьютером. По большей части весь ваш успех зависит в том насколько просто вы сможете объяснить компьютеру его задачу.

Начать стоит с того, что в любых языках программирования, пусть и по-разному будут использоваться одни и те же основные вещи, для примера далее я буду использовать Python, как самый простой язык для начинающих.

Самые основы программирования — это переменные – по своей сути ячейки хранящие определенные типы данных, обозначать переменные можно совершенно любыми латинскими буквами и специальными символами (например apple, numbers, a, b, _b, a3). Самое важное, ни в одном языке нельзя назвать переменную просто цифрой (1,2,3 и т. д.).

Переменные бывают разных типов – самые основные из них это string, integer, float (строка, целое число, дробное число). Такие типы данных могут хранить в себе лишь заданный тип данных. То есть в переменной типа int (сокращение от integer) может находиться только целое число, в string – только строка, в float – дробное число.

Пример присваивания переменной значения на Python:

Number = int(123)

Stroka = str(“привет”)

(Если быть честным, Python настолько странный язык, что тут можно писать даже Number = 123, абсолютно игнорируя присваивание определенного типа данных переменной)

В некоторых более старых языках (из приведенных в пример подойдет C++) вы можете наткнуться на shortint/longint или тип данных double, по своей сути это те же самые integer и float, просто в старых языках каждый тип данных выделял в памяти компьютера конкретное число бит/байт, чтобы не перегружать компьютер. Сейчас же это скорее редкость, т. к. вычислительные мощности наших компьютеров значительно выросли, и нагрузка в лишние пару байт не так сильно сказывается на производительности.

Над переменными можно производить совершенно различные математические операции, как и в обычной математике. Основные математические операции закреплены за своими специальными символами (+,-,*,/). Из важного: квадратный корень числа – sqrt(число), остаток от деления %, целочисленное деление Далее у нас есть логические выражения. В большинстве языков для проверок истинности/ложности выражения используется конструкция If, вот её пример на Python:

If number>10:
   number = number + 1
elif number<5:
   number = number – 5
else:
   number = 0

 Описать это можно как: если переменная number больше 10, тогда мы увеличиваем ее на 1. Если условие не выполняется идем дальше к следующему: если число меньше 5, тогда вычитаем из числа 5, и если и это условие не выполняется тогда приравниваем число к нулю.

Также неотъемлемой частью являются циклы, они бывают двух видов: for и while (Первый повторяет нашу программу ровно столько раз сколько мы попросим, второй до тех пор, пока выполняется условие). Пример на Python:

for i in range(5):
   number = number+1
while number<10:
   number = number+1

 Также важно упомянуть списки. Список это по своей сути переменная, хранящая в себе другие переменные, независимо от типа, на Python списки задаются так: list = [“a”,”b”,5,2.5,19]. Очень важно затронуть функции – своей сути они служат просто для сокращения количества символов в файле и общей читаемости кода. Функции делятся на встроенные в сам язык или одну из его дополнительных библиотек или пишутся вручную. Проще привести пример на Python, а после объяснить:

def Substract(a,b)
   return a-b
for i in range(5):
   print(Substract(i,2))

Здесь я использую встроенную функцию print() – по сути она просто выводит текст в файл вашего интерпретатора (Для Python интерпретатором является IDLE, по своей сути интерпретатор просто позволяет запускать ваш код, без создания целого приложения). Также здесь я объявляю функцию Substract в которую нужно ввести 2 значения, результатом этой функции (за возвращение результата отвечает return) является разность двух введенных чисел.

Честно говоря, это лишь малая часть от того, что необходимо знать для создания приложений. По большей части всё что нужно чтобы понять программирование – практика и стремление что-то сделать. Посоветую не пытаться делать сразу какие-то невероятные вещи - поизучайте синтаксис выбранного вами языка, попишите самый простой и банальный код, поизучайте то, как пишут другие люди на том же GitHub. Это даст вам опыт, и понимание логики программирования со временем, лучше, чем прочтение любого гайда. Любой навык нарабатывается.
